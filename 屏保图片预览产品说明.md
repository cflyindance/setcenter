# 屏保图片预览（列表缩略图大图）— 产品说明

> **实现位置**：`kiosk-theme-list.html` 中 `#imagePreviewModal` 及相关脚本。  
> **触发入口**：屏保主题卡片左侧 **竖版 / 横版** 缩略图（`.preview-thumb`）。  
> **数据范围**：读取当前主题在 `localStorage`（`kiosk_themes`）中的 `portraitImages`、`landscapeImages`（条目可为图片或 **视频** 元数据）；**只读预览**，不修改主题与素材。视频若仅存 `mediaRef`，打开预览前会经 **IndexedDB**（`KioskThemeMediaIdb`）解析为可播放地址。

---

## 1. 产品定位与目标

| 项目 | 说明 |
|------|------|
| **是什么** | 在屏保列表页，用户点击某主题的 **小图**，以全屏暗色遮罩形式 **查看该主题已有版式（仅竖、仅横、或竖+横）的素材**（**图片** 或 **视频**）；**仅当同时存在竖版与横版**时，提供竖/横 **Tab 快捷切换**；在合并后的幻灯序列中可上一张/下一张（多于 1 段时）。 |
| **不是什么** | 不是素材编辑、不是裁剪上传、不是下发配置；关闭预览后列表数据不变。 |
| **典型用户** | 运营核对「该主题实际带的图」是否与预期一致（含多图轮播）。 |

---

## 2. 业务逻辑

### 2.1 幻灯片序列（合并规则）

1. 打开预览时，根据当前主题的 `portraitImages`、`landscapeImages` 构建 **线性幻灯片数组** `currentPreviewSlides`。  
2. **顺序固定**：**先全部竖版**，再全部横版（与卡片上先竖后横的展示顺序一致）。  
3. 每张幻灯片对象包含：  
   - `src`：经 `previewMediaSrc` 解析（含视频在 IDB 就绪后的 `_displaySrc`）。  
   - `kind`：`image` 或 `video`（由 `isPreviewVideoMedia` 判定，与新建页写入的 `{ type:'video' }` / `data:video` 等一致）。  
   - `orientation`：文案「竖版」或「横版」。  
   - `groupIndex` / `groupCount`：在该版式内的序号 / 该版式共几段（用于底部副标题；计数单位对 **图** 为「张」，对 **视频** 为「段」）。  
4. 若两类素材 **均为空**，不打开预览（`openImagePreview` 内直接 `return`）。  
5. **打开前**：`hydrateThemeForPreview(theme)` 在支持 IDB 时，为带 `mediaRef` 的视频条目异步拉取 Blob 并挂 `_displaySrc`，再进入 `openWithTheme`；不支持 IDB 时直接尝试现有 `dataUrl` 等。  
6. **版式 Tab 显隐**：`previewShowsOrientationTabs()` 为真当且仅当 `portraitImages.length > 0` **且** `landscapeImages.length > 0`；为真时显示 `#previewOrientationSwitch` 并更新 Tab 态；否则隐藏整条 Tab，仅按合并序列浏览（仍为先竖后横的数组顺序，单一版式时数组仅含一段）。  
7. **起始位置**：由用户点击的缩略图决定——  
   - 点击 **竖版** 第 `imgIndex` 张 → 全局索引落在竖版范围内对应张；  
   - 点击 **横版** 第 `imgIndex` 张 → 全局索引 = 竖版张数 + 横版内对应张。  
   索引会做边界裁剪，避免越界。

### 2.2 与列表卡片的关系

- 缩略图在列表 `render()` 时生成，带 `data-theme-id`、`data-img-type`（`portrait` / `landscape`）、`data-img-index`。  
- 点击时读取上述属性，调用 `openImagePreview(themeId, imgType, imgIndex)`。  
- 主题必须在 `getThemes()` 中仍存在；若找不到主题，**不打开**预览。

### 2.3 关闭与清理

调用 `closePreview()` 时：隐藏模态、`stopPreviewVideo()`、撤销预览期间为视频创建的 Blob URL（`revokePreviewOpenBlobUrls`）、`previewImage.src` 置空、清空幻灯数组与索引、`previewPortraitCount` 归零，避免残留媒体占用内存或误显。

## 3. 界面与层级（交互设计要点）

| 层级（概念） | DOM 要点 | 作用 |
|--------------|----------|------|
| **底层遮罩** | `#previewBackdrop`：全屏、`z-0`、`bg-black/80` | 点击遮罩 **关闭预览**（`aria-label="关闭预览"`）。 |
| **中间层** | `pointer-events-none` 的满屏 flex 容器 | 用于居中内容；**自身不拦截点击**，避免误点「空白区」误关或误触（历史上曾用单层蒙层导致点「下一张」命中蒙层关闭的问题）。 |
| **可点控件** | 关闭按钮、**竖/横 Tab（仅双版式并存时显示 `#previewOrientationSwitch`）**、上一张/下一张、大图、计数条均带 `pointer-events-auto` 及较高 `z-index` | 保证只有明确控件响应点击。 |

预览模态与列表主内容 **同为 `z-50`**（与门店详情弹层同级）；后打开者会叠在上层，产品需注意避免同时强依赖两个全屏层（当前流程一般为二选一）。

## 4. 操作与交互一览

| 操作 | 行为 |
|------|------|
| **点击列表缩略图** | 打开预览，从对应竖/横图开始展示。 |
| **点击底层遮罩** | 关闭预览。 |
| **点击右上角 ×** | 关闭预览（`stopPropagation`，避免与遮罩重复逻辑冲突）。 |
| **竖版 / 横版 Tab** | **仅当主题同时有竖版与横版素材时**显示 Tab 条；**竖版**：将当前索引跳到 **第 1 张竖图**（全局索引 `0`）。**横版**：将当前索引跳到 **第 1 张横图**（全局索引 = 竖版张数）。若仅有单一版式，**不展示** Tab，大图区域上移边距略收（`mt-4` vs `mt-10`）。 |
| **上一张 / 下一张** | 仅在 **幻灯片总数 > 1** 时显示；在合并序列上 **循环**（首尾相接）切换。 |
| **键盘 Escape** | 关闭预览（仅当预览模态未隐藏时处理）。 |
| **键盘 ← / →** | 与上一张/下一张相同，**仅当多于 1 张** 时有效。 |

### 4.1 底部计数文案

格式：`当前序号 / 总数 · 竖版或横版 第 i/n 张` 或 `… 第 i/n 段`（实现中 **图片** 用「张」、**视频** 用「段」）。  

示例：`3 / 5 · 竖版 第 3/4 张`；若当前幻灯为视频则为 `… 第 2/3 段`。

若某版式 `groupCount` 为 0，副标题仅显示版式名（边界保护）。

## 5. 约束条件

| 约束 | 说明 |
|------|------|
| **必须有图** | `buildPreviewSlides` 得到的长度为 0 时不打开。 |
| **单张图** | 上一张/下一张按钮 **隐藏**；左右方向键 **不切换**。 |
| **仅竖或仅横** | **不显示**竖/横 Tab；仅展示该版式大图序列；可用箭头在序列内切换（若仅 1 张则仍无箭头）。 |
| **只读** | 预览内 **无**保存、替换、删除接口；不写入 `localStorage`。 |
| **主题存在性** | `themeId` 对应主题丢失时不打开。 |
| **全局键盘监听** | `keydown` 在文档上监听：通过 `previewModal.classList.contains('hidden')` 判断，避免在关闭状态下误处理方向键（仍可能与页内其他组件抢键，属已知局限）。 |

---

## 6. 与列表「⋯」菜单的关系

当前 `openImagePreview` **不因**主题上架或生效状态而禁止打开。列表 **⋯** 菜单规则以 **上架状态** 为准：**已上架** 时仅提供 **下架**（不可编辑、删除）；**缩略图预览**不受该限制，便于随时核对素材。

---

## 7. 技术依赖与素材形态

- 缩略图与大图同源：列表里已把 `portraitImages` / `landscapeImages` 以 `dataUrl` 或 URL 字符串写入 DOM。  
- 大图使用 `<img src="...">` 展示，`object-contain` 限制在视口内，避免撑破布局。  
- 若单图体积极大，可能影响低端设备内存与首帧绘制（产品可提示控制单图大小，与新建屏保页建议一致）。
